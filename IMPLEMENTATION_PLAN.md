## Stage 1: 未登录态审计与页面清单
**Goal**: 用 Chrome MCP 盘点未登录可见页面、组件与交互，形成可执行的克隆清单。  
**Success Criteria**:  
- 列出未登录可达的路由/入口（创意页、登录页、协议页、提示/示例弹窗）。  
- 每页输出组件树 + 状态矩阵（default/hover/active/disabled/loading/empty）+ 交互响应（跳转、弹窗、提示）。  
- 视觉基准表完成（字体/字号/颜色/圆角/阴影/间距/层级）并产出资源清单（图标/背景/示例图）。  
- 未登录拦截清单完成（点击触发登录提示的控件与行为）。  
**Tests**: 手动走查 + 清单勾选，覆盖 100% 可见与可触发的 UI 状态。  
**Status**: In Progress

## Stage 2: 未登录页面骨架与视觉基线
**Goal**: 搭建未登录页面骨架、全局样式与资源结构，保证布局和视觉基线一致。  
**Success Criteria**:  
- 路由与布局结构对齐目标站（创意页、登录页、协议页、弹窗容器）。  
- 设计变量落地（字体/颜色/圆角/阴影/间距），全局样式生效。  
- 复用基础组件（导航、工具条、按钮、下拉、Tooltip、弹窗）。  
- 静态资源按模块整理（如 `public/agent/*`）。  
**Tests**: 主要页面可正常渲染，无样式破版。  
**Status**: Not Started

## Stage 3: 未登录页面逐页克隆
**Goal**: 逐页实现未登录 UI 细节与交互响应，达到毫米级还原。  
**Success Criteria**:  
- 创意页：输入区、工具条、下拉选项、更多面板、示例弹窗、工具箱、案例列表/遮罩按钮。  
- 登录页：微信/手机号切换、验证码倒计时、协议链接跳转/新开、登录按钮与提示。  
- 协议页：排版层级、右对齐日期、滚动与留白。  
- 交互响应齐全：点击/hover/tooltip/弹窗/拦截提示。  
- 状态差异对齐：跨境/中文、白底主图选择等 UI 影响。  
**Tests**: 人工逐页对比 + 交互回归（手动）。  
**Status**: Not Started

## Stage 4: 细节打磨与响应式收口
**Goal**: 修正像素级偏差并完成多分辨率适配。  
**Success Criteria**:  
- 桌面与移动端布局对齐（断点、缩放、内容溢出处理）。  
- 视觉细节收敛（阴影、描边、渐变、图标对齐、透明度）。  
- UI 反馈齐全（hover/active/focus/disabled/loading）。  
**Tests**: 桌面/移动端截图对比 + 关键交互走查。  
**Status**: Not Started
